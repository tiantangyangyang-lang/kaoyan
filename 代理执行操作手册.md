# Claude Code 与 DeepSeek 4 Pro 执行操作手册

> 当前用户环境（2026-06-14）：Claude Code v2.1.156 已安装，并已配置 `deepseek-v4-pro` 后端。  
> 本方案使用 Claude Code 作为执行器、DeepSeek V4 Pro 作为模型；`cc-*` 表示确定性工程任务，`ds-*` 表示语义复核任务，不表示切换模型供应商。

## 1. 首次安装 Claude Code

在 PowerShell 中执行官方 WinGet 安装：

```powershell
winget install Anthropic.ClaudeCode
```

关闭并重新打开 PowerShell，然后验证：

```powershell
claude --version
claude doctor
```

首次进入 Claude Code 交互界面时：

```powershell
Set-Location D:\work\kaoyan
claude
```

若顶部显示 `deepseek-v4-pro`，说明 DeepSeek 后端已生效。执行带日志的任务时，应先用 `/exit` 离开交互界面，再运行本手册中的 `.cmd` 命令。

## 2. 不要使用的命令

处理真题时不要使用：

```text
claude --dangerously-skip-permissions
claude --permission-mode bypassPermissions
```

原因：两个真题来源仓库必须保持只读。默认权限模式会在高风险操作前请求确认。

## 3. 推荐的一键任务命令

所有命令从 `D:\work\kaoyan` 执行。

在 CMD 中切换目录使用：

```cmd
cd /d D:\work\kaoyan
```

`Set-Location` 仅适用于 PowerShell；如果提示符已经显示 `D:\work\kaoyan>`，无需再次切换目录。

```powershell
Set-Location D:\work\kaoyan
.\scripts\run-agent-task.cmd -Task cc-inventory -PlanOnly
```

该命令只让 Claude Code检查规范和输出执行计划，不修改文件。

### Claude Code：生成来源 inventory

```powershell
.\scripts\run-agent-task.cmd -Task cc-inventory
```

### Claude Code：转换数学一 2020

```powershell
.\scripts\run-agent-task.cmd -Task cc-math1-2020
```

### DeepSeek 4 Pro：复核数学一 2020

保持当前已经生效的 DeepSeek Claude Code 配置，执行：

```powershell
.\scripts\run-agent-task.cmd -Task ds-math1-2020
```

### Claude Code：转换数学二 2020

```powershell
.\scripts\run-agent-task.cmd -Task cc-math2-2020
```

### DeepSeek 4 Pro：复核数学二 2020


### Claude Code：一键连续跑队列

当前已经准备好的遗留结构修复队列：

```text
content\queues\math1-legacy-remaining.json
```

先只生成交接包，不实际调用外部 Claude Code：

```powershell
.\scripts\run-agent-queue.cmd -QueuePath .\content\queues\math1-legacy-remaining.json -PrepareOnly
```

在用户自己的本地 Claude Code shell 中连续执行整条队列：

```powershell
.\scripts\run-agent-queue.cmd -QueuePath .\content\queues\math1-legacy-remaining.json
```

更稳妥的入口是直接弹出一个外部 CMD 窗口：

```powershell
.\scripts\launch-agent-queue-external.cmd -QueuePath .\content\queues\math1-legacy-remaining.json
```

或者直接使用这个现成快捷入口：

```powershell
.\content\queues\launch-math1-legacy-remaining.cmd
```

默认遇到失败即停止；如果希望继续跑后续任务：

```powershell
.\scripts\run-agent-queue.cmd -QueuePath .\content\queues\math1-legacy-remaining.json -ContinueOnError
```

每次队列执行都会生成独立目录：

```text
content\reports\agent-queues\<timestamp>-<queue-name>\
```

优先查看：

1. `queue-summary.md`
2. `queue-log.md`
3. 每个子任务 `runDirectory` 下的 `final-summary.md`

脚本沿用当前 Claude Code 的模型供应商配置，不读取或修改 API Key。不要把 API Key 写入 `.md`、脚本、Git、截图或聊天记录。

## 4. 脚本做了什么

`scripts/run-agent-task.cmd` 会以单次 `ExecutionPolicy Bypass` 启动 `scripts/run-agent-task.ps1`，不永久修改系统执行策略。PowerShell 脚本随后会：

1. 检查 `claude` 是否安装。
2. 读取 `真题内容解析与代理处理规范.md` 和对应任务提示词。
3. 把当前批次需要的来源文件复制到本次运行目录中的 `source-mirror/`，不通过 `--add-dir` 暴露原始仓库。
4. 正式任务使用 `acceptEdits`，但只能在当前项目中写入；`-PlanOnly` 使用只读 plan 模式。
5. 沿用调用终端中已经生效的 Claude Code 模型后端；当前应显示 `deepseek-v4-pro`。
6. 为每次运行创建独立目录，保存完整终端日志、运行 manifest、实际 prompt 和代理结构化结果。
7. 运行前后对来源库做 SHA-256 快照；发现任何变化自动判定失败。
8. 校验代理必须提交的 `agent-result.json` 与 `agent-report.md`。
9. 生成 `final-summary.md`，明确任务成功、失败或阻塞。
10. 不修改调用终端已有的模型供应商与 API Key 配置。

代理正常情况下只会看到本次运行的来源副本，不需要访问原始仓库。如果出现任何访问或修改原始来源库的权限提示，必须拒绝。最终快照还会检测运行结束时仍存在的来源变化。

完整日志规范见 `代理运行日志规范.md`。

## 5. 运行后怎么看结果

每次执行会打印一个 Run directory，例如：

```text
D:\work\kaoyan\content\reports\agent-runs\20260614-210000-cc-math1-2020
```

按顺序查看：

1. `final-summary.md`：最终状态；失败或阻塞时不要开始下一批。
2. `agent-report.md`：代理做了什么、哪些内容需要人工确认。
3. `source-integrity.json`：必须显示 `passed: true`。
4. `output-integrity.json`：必须显示 `passed: true`，否则代理写入了任务范围外。
5. `agent-result.json`：核对读取数、生成数、警告、错误和检查结果。
6. `terminal.log`：出现问题时查看完整执行过程。

即使 `final-summary.md` 显示完成，真题内容也只能进入 `review/`，不能直接发布。

## 6. 推荐执行顺序

```text
cc-inventory -PlanOnly
  -> 人工查看计划
  -> cc-inventory
  -> 人工检查 inventory
  -> cc-math1-2020
  -> ds-math1-2020
  -> 人工逐题审核
  -> cc-math2-2020
  -> ds-math2-2020
  -> 人工逐题审核
```

不要一开始处理所有年份。数学一 2020 和数学二 2020 的试运行通过后，再增加年份。
