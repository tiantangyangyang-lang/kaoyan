# 真题内容解析与代理处理规范

> 版本：v0.1，2026-06-14  
> 目的：规定 Claude Code、DeepSeek 4 Pro 和人工审核者如何把现有数学一、数学二资料转换为网站可用的结构化题库。

## 1. 核心原则

两个原始仓库都是**只读来源库**，不是网站生产题库，也不是代理的工作目录。

- 数学一来源：`D:\work\Kaoyan-Math1-Papers`
- 数学二来源：`D:\work\Kaoyan-Math2-Papers`
- 所有转换结果、日志、异常报告和人工修订只写入：`D:\work\kaoyan`
- 实际代理运行时先复制本批所需文件到运行目录 `source-mirror/`；代理不得直接访问原始仓库。
- 禁止代理修改、重命名、移动、删除或格式化两个来源库中的任何文件。
- 禁止直接把来源库内容写入生产数据库。
- 禁止仅凭 OCR、DeepSeek 或 Claude Code 的输出将题目设置为 `published`。

## 3. 来源可信优先级

任何来源都不能跳过人工复核。出现冲突时，按以下顺序取证，而不是静默选择：

1. **原始 PDF / `*_origin.pdf` 的视觉内容**：用于人工确认题干、题号、公式、图片和解析。
2. **按年份整理的真题 Markdown**：作为题目切分和初始文本候选。
3. **解析 Markdown**：作为答案与解析候选。
4. **`content_list_v2.json` / `*_content_list.json`**：用于恢复内容块、页码、公式和图片关联。
5. **`layout.json` / `*_model.json`**：仅用于定位、调试和 OCR 追溯，不直接入库。
6. **AI 修复结果**：只能作为候选修订，必须保留修改前文本和理由。

若 Markdown 与 PDF 不一致，状态必须设为 `needs_human_review`。AI 不得自行决定哪个版本正确。

## 4. 输出目录与不可变来源记录

代理输出统一写入：

```text
D:\work\kaoyan\content\
  inventory/
    source-inventory.json
    source-anomalies.md
  staging/
    math1/{year}/
    math2/{year}/
  review/
    math1/{year}/
    math2/{year}/
  approved/
    math1/{year}/
    math2/{year}/
  reports/
    {batch-id}/
  fixtures/
```

每道题必须包含来源追踪字段：

```json
{
  "sourceRepo": "Kaoyan-Math1-Papers",
  "sourceRelativePaths": [
    "papers/2020年考研数学(一)真题.md",
    "solutions/2020年解析/2020年解析.md"
  ],
  "sourceCommit": "Git commit SHA",
  "sourceYear": 2020,
  "subjectCode": "math1",
  "sourcePageRefs": [],
  "transformVersion": "question-transform-v1",
  "reviewStatus": "needs_human_review"
}
```

只保存来源库相对路径，网站运行时不能依赖 `D:\work\...` 绝对路径。

来源 inventory 必须同时记录：

- 仓库 HEAD commit。
- 仓库是否 dirty。
- 文件是 tracked、modified 还是 untracked。
- 每个输入文件的 SHA-256。

未跟踪文件不能只用 `sourceCommit` 表示来源；必须依靠相对路径、SHA-256 和盘点时间追踪。

## 5. 标准处理流水线

```text
只读盘点
  -> 年份与文件配对
  -> 题目切分
  -> 题干/选项/答案/解析候选提取
  -> LaTeX 与图片引用规范化
  -> 知识点候选标注
  -> 自动质量检查
  -> DeepSeek 语义复核与异常标注
  -> 人工对照 PDF 审核
  -> approved JSON
  -> 数据库 dry-run
```

### 每批范围

- 正常批次：一个学科的一个年份，且最多 25 道题。
- 数学二 1987-2019 合并文件：必须先按年份拆分，再按单年份处理。
- 首次试运行：数学一 2020、数学二 2020。
- 异常挑战批次：数学一 2022、数学二 2021，用于验证 OCR 异常检测，不能作为第一批生产数据。
- 一个批次未通过人工审核前，不开始同来源的批量发布。

## 6. Claude Code 职责

Claude Code 只做可确定、可重复、可测试的工程处理：

- 生成来源 inventory、Git commit SHA、文件数量、年份覆盖和异常目录报告。
- 建立文件配对规则，将真题、解析、PDF、JSON 和图片关联到年份。
- 编写题目切分器、字段转换器、图片复制器和 JSON Schema 校验器。
- 生成稳定 ID，例如 `math1-2020-q01`。
- 检测重复年份、重复题号、缺失题号、缺失答案、断链图片和非法 JSON。
- 检查 Markdown/KaTeX 是否可解析，并输出失败位置。
- 生成批次差异报告、dry-run 报告和人工审核清单。

Claude Code 不负责：

- 判断数学答案是否正确。
- 修复含义不明确的公式。
- 决定知识点最终标签。
- 处理版权授权决策。
- 将内容直接发布到生产环境。

## 7. DeepSeek 4 Pro 职责

DeepSeek 只处理需要语义理解但仍需人工确认的内容工作：

- 根据题目结构候选切分题干、选项、答案与解析。
- 识别明显 OCR 噪声、错科目、缺失条件、公式疑似错误和解析题号错配。
- 将可确认的 LaTeX 风格统一为项目规范，但保留原文和修改说明。
- 为每题提出 1-3 个知识点候选，并说明证据。
- 比较真题与解析，输出冲突清单。
- 为人工审核生成简洁的“需要确认什么”说明。

DeepSeek 输出必须包含：

```text
原始来源：
候选结构化结果：
修改说明：
不确定项：
冲突项：
建议知识点：
置信度：
```

DeepSeek 不得：

- 在来源缺失时自行补写标准答案或详细解析。
- 把“看起来合理”当作“已核验正确”。
- 删除无法解析的内容。
- 修改来源库。
- 标记内容为已发布。

## 8. 自动质量门槛

进入 `review/` 前必须全部通过：

- 年份、学科、题号和题型均存在。
- 同一试卷稳定 ID 唯一。
- 选择题选项完整且答案属于合法选项。
- 题干、答案和解析保留来源路径。
- Markdown 与 JSON Schema 合法。
- KaTeX 测试无解析错误；无法支持的命令进入异常报告。
- 图片引用存在，且复制后的内容哈希一致。
- 不包含明显错科目标记，例如数学二文件标题出现“数学三”。
- 不包含已知高风险 OCR 模式且未标注，例如 `rx²`、拆散的 `l i m`、随机乱码。
- 所有 AI 修改都有原文、修改后文本和理由。

进入 `approved/` 还必须满足：

- 人工逐题对照 PDF 或可信来源完成审核。
- 答案和解析由人工确认。
- 知识点标签由人工确认。
- 内容使用授权状态明确。
- 审核人、审核时间和来源 commit 已记录。

## 9. 冲突与异常策略

- 同一年存在多个真题版本时，全部保留为候选，生成差异，不自动覆盖。
- 只有解析没有真题时，标记 `missing_paper`，禁止合成题干。
- 只有真题没有解析时，允许题目进入审核，但答案/解析状态必须为缺失。
- 错科目目录或文件进入隔离清单，不参与转换。
- 题号切分失败时保留整段原文，状态设为 `segmentation_failed`。
- 公式无法确认时保留原始 LaTeX 和页面截图引用，状态设为 `formula_needs_review`。
- 代理任务失败时只重跑当前批次，不得覆盖已审核批次。

## 10. 许可与发布边界

- 数学一仓库标明 CC BY-NC-SA 4.0，其中 `NC` 禁止默认商业使用；网站若商业化，必须取得额外授权或更换内容来源。
- 数学二仓库未发现许可证，默认视为“授权不明确”，不得直接公开发布。
- 真题版权归属不能仅由 GitHub 仓库许可证决定。
- 开发和内部验证可使用脱敏、少量样例；公开上线前必须由项目负责人确认内容来源和使用方式。

## 11. 代理任务统一模板

```text
批次 ID：
目标学科与年份：
只读输入根目录：
允许读取的相对路径：
允许写入的工作区路径：
禁止修改的路径：
输入 commit SHA：
转换 Schema 版本：
本批最大题数：
必须运行的检查：
人工审核门槛：
输出文件清单：
异常与未决项：
```

每个代理任务结束后必须报告读取文件数、生成题目数、跳过数、异常数、输出路径和检查结果。

每次实际运行还必须遵循 `代理运行日志规范.md`，生成独立运行目录、来源前后快照、`agent-result.json`、`agent-report.md` 和 `final-summary.md`。缺少任一强制结果文件时，批次视为失败。
